# 🛠️ 제작가이드 모듈 관리 규칙 (guide_modules)

이 문서는 `generate_combined_html.py` 스크립트를 사용하여 단색/별색 가이드 이미지를 HTML로 생성하는 방법을 설명합니다.

## 1. 📂 폴더 구조

스크립트가 파일을 정확히 감지하려면 아래 폴더 구조를 유지해야 합니다.

. ├── guide_modules/ │ ├── mono_case/ <-- 단색 (Monochrome) 가이드 이미지 │ └── spot_case/ <-- 별색 (Spot Color) 가이드 이미지 └── generate_combined_html.py


---

## 2. 🔑 핵심 규칙: 파일명 지정

스크립트는 파일명 내 **가장 앞에 나오는 숫자**를 기준으로 가이드 이미지의 순서를 자동 정렬합니다.

### A. 순번 기준 (필수)

* **HTML에 나열될 순서**를 결정하는 숫자는 반드시 파일명 **가장 앞에 위치**해야 합니다.
* 이 숫자는 **중복될 수 없으며** (각 파일은 고유한 순번을 가져야 함), 순차적으로 증가해야 합니다.

| 권장 (O) | 비권장/오류 가능성 (X) |
| :--- | :--- |
| `01_description.jpg` | `A_01_description.jpg` (문자열이 먼저 와서 정렬이 안 됨) |
| `02.jpg` | `1.jpg`와 `10.jpg`가 섞여서 정렬될 때 순서 혼동 가능성 있음 |
| `10_detail.jpg` | `2_detail.jpg`가 `10_detail.jpg`보다 뒤에 나열될 수 있음 |

### B. 숫자 형식 (권장)

* 순번은 두 자릿수 (**Zero Padding**)를 사용하여 `01`부터 시작하는 것을 강력히 권장합니다. (예: `01`, `02`, `10`, `11`, ...)

---

## 3. 🚨 정렬 원리 및 주의사항

스크립트의 정렬 로직은 **파일명에서 가장 처음 발견되는 숫자**를 **정렬 키**로 사용합니다.

### A. 정렬 키 추출 로직

| 파일명 | 정렬 키 (스크립트가 인식하는 값) |
| :--- | :--- |
| `**01**.detail.ver1.jpg` | `1` |
| `**02**.guide.ver2.jpg` | `2` |
| `**10**.step.final.jpg` | `10` |

### B. 중복 순번 사용 금지 (최대 주의)

만약 **두 파일의 정렬 키가 동일**한 경우 (예: `01_case_A.jpg`와 `01_case_B.jpg`), 스크립트는 **안정 정렬**을 수행하여 두 파일의 순서가 **파일 시스템이 읽어온 순서**대로 결정됩니다.

**이는 예측 불가능하므로, 하나의 폴더 안에서 첫 번째 숫자는 절대 중복되어서는 안 됩니다.**

### C. 파일 형식

* 파일 형식은 반드시 **`.jpg`** (대소문자 무관)여야 합니다.

---

## 4. ▶️ HTML 생성 방법

1.  `guide_modules/mono_case` 또는 `guide_modules/spot_case` 폴더에 위 규칙에 맞는 `.jpg` 파일을 추가/수정합니다.
2.  스크립트를 실행합니다.

    ```bash
    python generate_combined_html.py
    ```

3.  `output/final_guide.html` 파일이 생성되었는지 확인합니다.
